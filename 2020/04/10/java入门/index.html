    简介

    • Java最早是由SUN公司（已被Oracle收购）的James Gosling在上个世纪90年代初开发的一种编程语言，最初被命名为Oak，目标是针对小型家电设备的嵌入式应用，结果市场没啥反响。在1995年以Java的名称正式发布，传说是Oak已经被人注册了，因此SUN注册了Java这个商标。
    • 编译型语言：C、C++ 等，运行前编译，源码直接编译成机器码执行，不同的平台(x86,ARM等)CPU的指令集不同，需要编译出每一种平台对应的机器码。
    • 解释型语言：Python 、Ruby、JavaScript 等是解释型语言，在运行的时候进行编译，所以，由解释器直接加载源码后运行，代价是运行效率低于编译型语言（因为解释器要首先将源码翻译成机器码）
    • Java介于编译型语言和解释型语言之间。
    • java是将源代码编译成一种“字节码”，类似于抽象的CPU指令，然后针对不同平台编写java虚拟机，不同平台的虚拟机负责加载字节码并执行。实现了“一次编写，到处运行”，为了保证不同平台，不同公司开发的虚拟机能正确执行JAVA字节码，SUN公司制定了一系列的java虚拟机规范，JVM的兼容性很好，低版本的JAVA字节码可以正常运行在高版本的JVM上。

      JVM：一种能够运行Java字节码（Java bytecode）的虚拟机。

    产品版本

    随着JAVA的发展，现在JAVA分为3个版本，面向不同的需求。

    简称 全称 特点
    Java SE java standerd Edition JVM和标准库
    Java EE java Enterprise Edition 在SE的基础上加了大量的API和库,方便开发Web 应用 数据库 消息服务等
    Java ME java Micro Editon 略,正在消失中

    从上表看出Java SE 是 JAVA的核心。

    名称解释

    JDK

    • JDK（Java Development Kit） 是将Java 源代码编译成Java字节码的工具，也是JAVA开发工具;JDK除了包含JRE、还提供了编译器(compile)、调试器等开发工具。
    • JRE: Java Runtime Environment,JRE是Java 运行Java字节码的运行环境。

    JSR

    • Java Specification Request
    • 为了保证Java语言的规范性，SUN公司建立了JSR规范;凡是想给Java平台加一个功能，要先定要好接口。比如说增加访问数据库的功能，大家要先创建一个JSR规范，定义好接口，这样，各个数据库厂商都按照规范写出Java驱动程序，开发者就不用担心自己写的数据库代码在MySQL上能跑，却不能跑在PostgreSQL上。
      所以JSR是一系列的规范，从JVM的内存模型到Web程序接口，全部都标准化了。

    JCP

    • Java Community Process
    • JCP就是负责审核JSR的组织。JSR规范发布时，为了让大家参考，同时还要发布一个“参考实现”，以及一个兼容性测试套件:
    • RI：Reference Implementation
    • TCK: Technology Compatibility Kit

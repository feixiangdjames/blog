    整数运算

    • 整数运算结果永远精确。
    • 预算结果的类型会自动提升。
    • 强制转型，可能会损失。

      整数

    • 依照四则预算规则，如果变量是整型，则结果也是整形；
    • 如果定义的变量是整型，则计算的算子都只能是整型。如果定义的变量是浮点型，则算子可以有浮点型。
    • 被除数是0，不会影响编译；JVM运行字节码的时候会报错。
    • ++ 和 – 运算符，这里要注意++的位置，在变量之前，是先加后引用；在变量之后是先引用，之后自身加1.
    • += 相当于 n+=1==>n=n+1;

      注意：java 中的整数是有符号的整数。在计算机中的整数都是以二进制补码形式进行存储的。

    |类型|说明|
    |正数|源码，补码，反码都是该数字本身；整数的符号位是0|
    |负数|负数的符号位是1，计算机的二进制代码是补码；想得到这个数字的源码，先获取反码，然后+1，获得补码|
    |0|+0-0|都是0|

    • 11111111111111111111111111111111
    • 先取反码 获得1—31个0，然后补码-》末尾+1，结果就是1–0001,高位的1代表负数，就是-1.

    位运算

    • 首先位对齐，然后依次对每位进行运算。
    • 1 & 0 // 结果是0
    • 1 | 0 ///结果是1

    异或运算

    • 位相等的时候是 0,位不同的是1

    非运算

    • ~1=0
    • ~0=1

    浮点数运算

    • 浮点数无法精确表示。
    • 计算有误差。
    • 计算过程中，整型会自动提升到浮点型。
    • 以下情况类型不自动提升
    • 浮点数除0，是有别于整数的。
    • 浮点型强制转型，并不会四舍五入，而是直接扔掉小数位位数。
    • 浮点型四舍五入的技巧是加0.5
    • 超出整型范围的浮点型转型，会便问最大值

    boolean 运算

    • 短路运算符 && 和 ||
    • &&
      boolean and =false&&(5/0>0);//并不会报错,因为第一个值计算为false
      boolean and=false && (5/0>0);//false
      boolean or=true||(5/0>0);//true,有一个ture就不会运行后面的代码

    访问权限

    总结
    JAVA内建的访问权限包括public、protected、private、package
    局部变量的作用域从变量声明开始，到一个块结束
    一个JAVA文件只能包含一个public class，但可以包含多个非public class

    • JAVA的类、接口、字段和方法都可以设置访问权限

    • 访问权限有public、 protected、private 和 package四种。

      定义为public的field、method可以被其他类访问
      定义为private的field、method无法被其他类访问

      private

    • private 访问权限限定在class的内部，与方法的声明顺序无关

    • 建议把public放在private前面，因为public对外提供了方法，首先应该被关注。

    • 定义为private 的class 无法被其他类访问

    • 访问private class 被限定在外层class 的内部

      定义在一个class的内部的class称为inner class

      protected

      定义为protected的字段和方法可以被子类访问

      package

      包作用域是指一个类允许访问同一个package的。

    • 没有public、private修饰的class

    • 没有public 、protected、private修饰的字段和方法。

    • 包没有继承关系，package权限 包名必须一致。

      最佳实践
      如果不确定是否需要public ，就不声明为public ，即尽可以能少暴露对外方法
      因为权限从小改大容易，从大改小要非常小心

      局部变量

    • 在方法内部定义的变量称为局部变量，方法名也是局部变量。

    • 局部变量作用域从变量声明处开始到对应的块结束。

    • 尽可能的把局部变量作用域缩小。

    • 尽可能的延后声明局部变量.

      注意：在一个java文件中，只能定义一个public class,但是可以定义多个非
      public class.并且保存在同一个java 文件中。

      final

    • final 修饰方法，阻止覆写。

    • final 修饰类，不能继承。

    • final field 不能重新赋值。

      classpath

    • classpath 是一个环境变量

    • classpath 指示JVM如何搜索class

    • classpath 设置的搜索路径与操作系统相关

    • window中;fenge

    • mac OS 是使用： 分割

      例如设置的classpash是 .;C::\work\project1\bin;c:\shared

    • JVM在加载com.feiyangedu.Hello这个类时，依次查找

    1. 当前目录 \com\feiyangedu\Hello.class
    2. C:\work\project1\bin\com\feiyangedu\Hello.class
    3. C:\shared\com\feiyangedu\Hello.class

    • 在某个路径下找到了，就不继续了

    • 如果都没有就报错

      classpath的设置方法：

    • 直接在系统环境中设置classpath 环境变量（不推荐）

    • 在启动JVM时设置classpath变量（推荐）

    • java -classpath C:\work\bin;C:\shared com.feiyangedu.Hello

    • java -cp C:\work\bin;c:\shared com.feiyangedu.Hello

    • 没有设置环境变量，也没有设置-CP参数，默认的classpath为.当前目录

    • 在Eclips中运行JAVA程序，eclipse自动传入-cp参数是当前工程的bin目录和引入的jar

      jar包

    • jar包是zip格式的压缩文件，包含若干.class 文件中。

    • jar 包相当于目录

    • classpath可以包含jar文件 C:\WORK\bin\all.jar

    • 查找com.feiyangedu.Hello类将在C:\work\bin\all.jar文件中所有

      jar包生成

    • 使用JDK自带的jar命令

    • 使用构建工具如Maven等

      jar包的其他功能

    • jar包可以包含一个特殊的/META-INF/MANIFEST.MF文件

    • MANIFEST.MF是纯文本，可以指定Main-Class和其他信息

    • jar包可以包含其他jar包

      JDK的class

    • JVM运行时会自动加载JDK自带的class

    • JDK自带的class被打包在rt.jar

    • 不需要在classpath中引用rt.jar

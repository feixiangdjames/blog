    |总结|
    |—|
    |JDK的包装类型可以把基本类型包装为class|
    |自动装箱和自动拆箱是编译器完成的|
    |装箱和拆箱会以向执行效率|
    |拆箱是可能会发生NullPonterException||||||||||

    wrapper

    • 定义一个Integer类，包含一个实例字段int
    • 可以把lnteger看作一个int的包装类型（wrapper）

    JDK为每种基本类型都创建了对应的包装类型：

    基本类型 对应的引用类型
    boolean Boolean
    byte Byte
    short Short
    int lnterger
    long Long
    float Float
    double Double
    char Character

    int lnteger和String的相互转换

    特别注意lnteger.getInteger(String)是从系统环境中读取系统变量。

    • 对于基本类型和它对应包装类型，编译器可以自动在int和lnteger之间转型：

    • 自动装箱： auto boxing
      int->lnteger

    • 自动拆箱：aauto unboxing
      integer->int

    • auto boxing ,auto unboxing 只发生在编译阶段。

    • boxing,unboxing 影响执行效率

    • 编译后的class代码时严格区分基本类型和引用类型的

    • integer->int 执行时可能会报错

    • 整数和浮点数包装类型继承自Number
      graph BT
a(Byte)-->n(Number)
i(integer)-->n(Number)
f(Float)-->n(Number)
n-->o(Object)

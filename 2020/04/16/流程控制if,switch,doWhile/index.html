    if

    • 只有一条执行语句的时候，可以省略{}。
    • 不推荐省略，因为，省略了{},当你想修改代码的时候，
      执行大于一条语句的时候，会忘记加{}。
    • else 和 if else 并不是必须的。
    • 注意边界判断。
    • 浮点数用 == 判断不靠谱.== 对引用类型来是判断是否指向同一对象。
    • 对于浮点数利用差值小于狗哥临界值判断，例如下面的例子。

      if / else if /else

    graph TD
a[int=70]-->b{n>=90?}
b-->|N|c{n>=60?}
c--N-->h[fail]
c--Y-->g[success]
h-->x

g-->x[end]

    equals()

    • 如果想判断引用类型的值是否相等，要使用equals方法。
    • 如果变量是null,调用equals()，会有运行时报错。因为 null 不指向任何对象。

    switch case

    最重要的是不要忘记break

    switch (表达式) 跳转到怕匹配的case 结果；
    继续执行case 结果的后续语句
    遇到break结束执行
    在没有遇到任何匹配项目的时候，还可以执行一个可选的default
    case 没有花括号，以：开头的,：后面可以一条或多条语句。
    switch de 计算结果必须是整型，字符串、枚举型
    建议打开fall-through和missing default 警告。
    浮点不能做精确的==判断

    循环

    while 循环

    • while 首先判断条件
    • 条件满足循环，条件不满足退出，可能一次都不执行。

    do-while 循环

    • 是至少执行一次,先执行循环体，然后再判断循环条件。

    for

    • 计数器的初始值
    • 每次的循环条件
    • 更新计数器
    • 条件判断，是否进行下一次循环
    graph TD
a[计数器初始]-->b[判断循环条件]
b--TRUE-->m[跳出循环]
b--FALSE-->c[执行循环体]
c-->d[更新计数器]
d-->b

    初始化计数器的代码和第一次条件判断，总是执行的

    for each 循环

    • for each 循环能够便利数组和可迭代的数组类型，List\Map等
    • for each 循环 无法指定遍历顺序。
    • 无法获得数组索引。

      在不需要索引的情况的，我们总是使用 for each 循环。

    break

    • 跳出当前的循环体。

    continue

    • 提前结束当前循环，更新计数器后，开始下一轮循环。

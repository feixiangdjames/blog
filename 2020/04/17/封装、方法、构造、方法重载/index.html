    数据封装

    • 一个calss可以包含多个field。
    • 直接把field用public 暴漏给外部，可能会破坏封装。
    • praivate 修饰 field更加的安全.可以检查传入的参数值。
    • 用private 修饰 field 可以拒绝外部访问.
    • 定义public 方法 可以简介修改field.

    • 当使用private 定义 field 。外部代码（不属于当前类的）
      使用变量名.field会编译错误。

    • 使用public 方法封装数据访问，更加的安全，可以为访问
      行为执行行为规范。所以对于private 只可以使用变量.方法名()进行访问。

    方法特点

    方法
    可以让外部代码安全的访问实例字段
    一组执行语句
    遇到return返回
    void 表示不返回任何值（区别于返回null）
    外部代码通过public 方法操作实例
    内部代码可以调用private 方法

    方法定义

    • 方法修饰符：

      1. public 外部可以访问,eclipe会列举出所有的public方法，不包括private 方法。
      2. private 仅内部可以访问，多个public方法访问同一个public 方法，提高代码复用。

    • 方法返回值

    1. void 没有返回值
    2. 有返回值，定义返回值类型，比如Sring,int 等

    • 方法名称
      驼峰命名法

    • 方法的参数列表
      如果没有参数，可以为空。
      如果有多个参数，需要用逗号隔开,单一参数是由 参数类型和参数名称组成

    • 方法可以包含一行或多行语句，方法的返回值是根据return进行实现的

    this

    • 方法内部可以使用隐式变量this,this永远指向当前实例。
      this.field可以访问类的字段(实例字段)

    • 在不引起歧义的情况下，可以省略this

    • 是否创建对象的时候，就可以直接初始化field呢？

    实现这样功能的方法，就是类的构造方法，构造方法可以用于构造实例。

    默认构造方法

    • 如果一个类没有定义构造方法，编译器会自动生成一个默认构造方法：
      • 无参数
      • 无执行语句

    实例初始化顺序

    • 先初始化字段。是的，可以在定义field时候，就进行初始化。
    • 没有赋值的field，初始化为默认值；
      int 0
      boolean false
      引用类型 null
    • 执行构造方法代码

    多构造方法

    • 可以定义多个构造方法，编译器通过构造方法的参数数量、位置和类型区分
    • 我们可以在一个构造方法中调用另一个构造方法。this();

    Overload

    Overload 更像一种编写思路

    定义

    • 多个方法名相同
    • 参数不同：个数，位置，类型
    • 方法返回值相同

    总结

    • 重载方法应该完成相同的功能，参考String的indexOf();
    • 重载方法依靠参数类型和数量区分
      1. indexOf(String str)
      2. indexOf(String str,int fromlndex)
    • 不要去交互参数顺序，来实现重载；虽然可以执行，但是没有意义
    • 重载方法返回值类型应该相同

    类的构造方法就是一种 Overload的表现.

    总结
    对称加密算法使用同一个密钥进行加密和解密
    常用算法：DES/AES/IDEA等
    密钥长度由算法设计决定，AES的密钥长度是128/192/256
    使用256位加密需要修改JDK的policy文件
    使用堆成加密算法需要指定：算法名称/工作模式/填充模式
    PBE算法通过用户口令和随机Salt计算Key 然后加密
    Key通过口令和随机Salt计算得出，提高安全性
    PBE算法内部使用的仍然是标准堆成加密算法

    对称加密算法

    加密和解密使用同一个密钥

    • 例如：WinRAR

    加密：encrypt(key,message)->s
    解密：decrypt(key,s)-> message

    常用的对称加密算法：DES,AES,IDEA
    密钥长度直接决定了的加密的强度。
    DES只有56/64位，可以暴力破击，0已经不建议使用了。

    JDK并没有包含所有的工作模式和填充模式，通常我们只需要使用常用的就可以了。

    PBE 算法

    PBE： Password Based Encryption

    • 由用户输入口令，采用随机数杂凑计算出密钥再进行加密
    • Password:用户口令
    • Salt:随机生成的byte[]
    • key:generate(byte[] aslt,String password)

    AES vs PBE

    标准的AES算法中，key 是自动随机生成。
    PBE算法中，key是用户的输入的口令加随机数凑出。

    如果把salt固定，就得到一个通用的口令加密软件。

    如果把随机salt存储在U盘，就得到了一个“口令” + USB Key加密软件。
    即时用户使用非常弱的口令，没有USB key 仍然无法破解。

    ReentrantLock可以替代synchronized
    ReentrantLock获取锁更安全
    必须使用try…finally保证正确获取和释放锁
    使用ReadWriteLock可以提高读取效率
    ReadWriteLock只允许一个线程写入
    ReadWriteLock允许多个线程同时读取
    ReadWriteLock适合读多写少的场景
    Condition 可以替代wait/notify
    Condition 对象必须从ReentrantLock对象中获取
    ReetrantLock+Condition 可以替代 synchronized+wait/notify
    使用java.util.concurrent提供的Blocking集合可以简化多线程编程
    多线程同时访问Blocking集合是安全的
    尽量使用JDK 提供的concurrent集合，避免自己编写同步代码

    ReentrantLock

    线程同步：

    • 多线程读写竞争资源需要同步
    • Java语言提供了 synchronized / wait /notify
    • 编写多线程同步很困难

    为了解决这个问题，JDK提供了 java.util.concurrent包

    • 更高级的同步功能
    • 简化多线程程序的编写
    • JDK>=1.5

    java.util.locks.ReentrantLock用于替代synchronized加锁：

    • synchronized是java语言层面提供的，不需要考虑异常。
    • ReentrantLock 是一个普通的java class，必须用 try finally 来保证锁能够正确释放。

    ReentrantLock使用

    ReentrantLock 特点

    • 可重入锁，一个线程可多次获取同一个锁
    • lock()方法可获取锁
    • tryLock()方法可尝试获取锁并指定超时

    ReadWriteLock适用条件：

    • 同一个实例，有大量线程读取，仅有少数线程修改

    Condition

    java.util.locks.ReentrantLock 用于替代synchronized加锁：

    synchronized 可以配合 wait/notify 在线程条件不满足的时候等待.
    实现 wait/notify 就要使用Condition对象：

    Condition

    Condition.await/signal /signalAll 原理和 wait/notify /notifyAll 一致
    await()会释放当前锁，进入等待状态
    signalAll() 会唤醒所有等待线程
    唤醒线程从 await()返回后需要重新获得锁.

    Concurrent 集合

    Blocking Queue

    ReentrantLock + Condition 实现 Blocking Queue：

    java.util.concurrent 提供了线程安全的Blocking集合：

    • ArrayBlockingQueue

    • 多线程

    java.util.concurrent 提供了线程安全的Blocking集合：

    Interface Non-tread safe Thead safe
    List ArrayList CopyOnWriteArrayList
    Map HashMap ConcurentHashMap
    Set HashSet,TreeSet ConpyOnWriteArraySet
    Queue ArrayDeque,LinkedList ArrayBlockingQueue,LinkedBlockingQueue
    Deque ArrayDeque,LinkedList LinkedBlockingDeque

    java.util.Collections 工具类还提供了旧的线程安全集合转换器：

    • Map unsafeMap=new HashMap();
    • Map threadSafeMap=Collections.synchronizedMap(unsafeMap);

    以上代码实际上是一个包装类，所有的读写方法都用synchronized加锁，这样获得的线程
    安全的集合性能比concurrent低很多，现在基本不推荐这种方法。

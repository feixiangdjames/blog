    Annotaion

    • 注解是放在JAVA 源码的class,function,field,参数前的一种标签。
    • 注解本身对代码逻辑没有任何影响
    • 如何使用注解由工具决定

    常见注解

    • @Verride:
    • @Deprecated:方法作废标记
    • @SuppressWarnings：忽略方法内部错误

    注解可以定义配置参数：

    • 配置参数由注解类型定义

    • 配置参数包括：

      所有基本类型
      String
      枚举类型
      数组 String,枚举类型，基本类型

    • 配置参数必须是常量

    • 缺少配置参数将使用默认值
      如果只写常量，相当于忽略了value
      @Check(value=99) === @Check(99)

    • 如果只写注解相当于全部使用默认值

    定义注解

    使用 @interface定义Annotaion

    • 注解的参数类似无参数方式
    • 可以设定一个默认值
    • 把最常用的参数命名为value

    元注解

    @Target

    使用@Target定义Annotation可以被应用于源码的哪个位置：

    • 类或接口：ElementType.TYPE
    • 字段： ElementType.FIELD
    • 方法： ElementType.METHOD
    • 构造方法：ElementType.CONSTRUCTOR
    • 方法参数：ElementType.PARAMETER

    @Retention

    使用@Retention定义Annotation的生命周期：

    • 仅编译：RetentionPolicy.SOURCE
    • 仅class文件:RetentionPolicy.CLASS
    • 运行时：RetentionPolicy.RUNTIME
    • 如果@Retention不存在，则该Annotation默认为CLASS
    • 通常自定义的Annotation都是RUNTIME

    Annotaion的生命周期

    • RetentionPolicy.SOURCE : 编译时直接丢弃，只在源码中使用
    • RetentionPolicy.CLASS:仅存在class文件中
    • RetentionPolicy.RUNTIME：在运行期可以读取该Annotation

    @Repeatable定义Annotation是否可以重复

    如果使用 @Repeatable修饰一个Annotation，则这个Annotation可以反复使用

    @Inherted

    子类是否可以继承父类的Annotaiton

    • 针对@Target为TYPE类型的Annotation
    • 仅针对class的继承
    • 对Interface的继承无效

    定义个Annotation

    • 用@interface定义注解
    • 用元注解（meta annotation）配置注解
      • Target :必须设置
      • Retention:一般设置为RUNTIME
      • 通常不必写@Inherted，@Repeatable
      • 定义注解参数和默认值

    处理注解

    • RetentionPolicy.SOURCE : 编译时直接丢弃
    • RetentionPolicy.CLASS:仅存在class文件中
    • RetentionPolicy.RUNTIME：在运行期可以读取该Annotation

    SOURCE 只在编译器中有效，编译时就丢弃了

    读取RUNTIME类型的注解：

    • Annotaion 也是class
    • 所有Annotation继承自java.lang.annotation.Annotaion
    • 使用反射API

    反射API 读取Annotation:

    • Class.isAnnotationPresent(Class)
    • Field.isAnnotationPresent(Class)
    • Method.isAnnotationPresent(Class)
    • Constructor.isAnnotationPresent(Class)

    总结
    JVM为每个加载的class创建对应的Class实例来保存 class所有信息
    获取一个class对应的Class实例后，就可以获取该class的所有信息
    通过Class实例获取class信息的方法称为反射
    JVM总是动态加载class,可以在运行期更具条件控制加载class

    class

    java 除基本类型外都是class（包括interface）

    • Sring
    • Object
    • Runnable
    • Exception
      …

    class的本质是数据类型。
    无继承关系的数据类型无法赋值。JAVA 定义了一种强类型的数据关系。
    并且，编译器会检查数据类型，不符合数据类型要求的，编译器会报错。

    class/interface 的数据类型是Class

    • 每加载一个class，JVM就为其创建一个Class类型的实例，并关联起来
    graph TD
a(加载String类:
    读取String.class文件)-->b(为String类创建一个Class实例:
    class cls=new ClassString)
    • JVM持有的每个Class实例都指向一个数据类型(class / interface)
    Class实例
    name “java.lang.String” Sring
    name “java.util.Random” Random
    name “java.lang.Runnable” Runnable
    • 一个Class实例包含了该class的完整实例。
    Class实例
    name “java.lang.String”
    package “java.lang”
    super java.lang.Object
    interface CharSequence,…
    field value[],hash,..
    method indexOf,valueOf

    JVM 为每个加载的class创建对应的Class实例，并在实例中保存该class的所有信息
    如果获取了某个Class实例，则可以获取到该实例对应的class的所有信息
    通过Class实例获取的class信息的方式称为反射(Refection)

    Refection

    如何获取一个Class实例？

    • Type.class
    • getClass()
    • Class.forName()
    • JVM中，一个class对应唯一的一个Class实例；也就是说Class实例在JVM中式唯一的。
      • 可以用==比较两个Class实例

    Class实例比较 VS instanceof

    Class实例比较只能精确判断。所有在判断数据类型的时候，我们使用instanceof，因为
    面向抽象编程的时候，我们不关心具体的子类型。

    Refection目的

    反射的目的式当获得某个Object实例时，我们可以获取该Object的class信息

    从Class实例获取的calss信息：

    • getName();
    • getSimpleName();
    • getPackage();
      从Class实例判断class类型：
    • isInterface()
    • isEnum()
    • isArray()
    • isPrimitive()

    创建class 实例

    • newInstance()
    • 局限性，我们只能调用不带参数的String类默认构造方法

    动态加载

    利用JVM动态加载class的特性：

    • 可以在运行期根据条件加载不同的实现类
